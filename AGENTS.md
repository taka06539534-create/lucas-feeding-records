# Codex 项目指令

本项目用于整理和分析新生儿喂养护理记录。处理本项目内的喂养日报、可视化报告或 `baby-feeding-summarize` 相关任务时，遵守以下默认要求。

## 宝宝基础信息

- 生成日报或分析记录前，先读取根目录的 `baby-info.md`。
- 报告中需结合 `baby-info.md` 里的宝宝基本信息，例如性别、出生日期、记录日龄和出生第几天。
- 如果 `baby-info.md` 缺失、字段不完整或日期格式无法可靠解析，不要编造；在报告的信息缺口中说明，并在需要时用“疑似/按可能值估算”标注。
- 若记录日期与 `baby-info.md` 可解析出生日期都明确，应基于二者计算日龄；不要使用系统当前日期代替记录日期。

## 可视化文本

- 生成 SVG、HTML、PNG 或其他可视化视图时，注意长文本换行或压缩。
- 图表中的 `summary`、`comparisons`、图例、卡片说明等字段优先使用短句；详细解释放到 Markdown 正文中。
- 如果渲染器支持换行，长文本应主动拆成多行；如果渲染器不支持自动换行，应缩短图表字段，避免文本超出容器。
- 生成后检查可视化文本长度和展示效果；发现可能溢出的文本时，先调整为短句再交付。

## README 同步

- 每次新增、更新、重生成或删除喂养记录、日报目录、结构化数据或可视化文件时，必须同步更新根目录 `README.md`。
- `README.md` 的“喂养记录索引”至少要包含记录日期、日报入口 `index.md`、结构化数据、可视化视图和原始输入文件。
- 如果同一天存在多个报告目录，或报告目录因重名自动追加序号，应在 `README.md` 中明确列出实际入口，避免索引失效。
